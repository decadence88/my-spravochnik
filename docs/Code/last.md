=== ":: — (type cast)"
    <font size="5">**Приведение типа данных в PostgreSQL**</font>

    Используется, чтобы принудительно преобразовать значение из одного типа данных в другой (например, текст в число или текст в дату).
    
    **Примеры базового синтаксиса:**
    
    * `'2026-07-01'::date` — превращает строку в формат даты.
    * `'123'::int` — превращает строку в целое число.
    * `45.67::int` — округляет дробное число до целого (получится `46`).
    
    **💡 Живой пример в коде:**
    Представь, что в базе возраст хранится в виде текста, а нам нужно найти всех, кто младше 30 лет, и вывести дату их регистрации в красивом формате:
    
    ```sql
    SELECT 
        user_id,
        age_text::int AS user_age,
        registration_date_text::date AS registered_at
    FROM S080TGL.USERS
    WHERE age_text::int < 30;
    ```

=== "|| — Объединение строк"
    <font size="5">**Соединение текста в PostgreSQL**</font>

    Используется, чтобы склеить значения нескольких колонок или обычный текст в одну единую строчку.
    
    **Важно:** Если при склеивании попадается значение `NULL` (пустота), то весь результат превратится в `NULL`. Чтобы этого избежать, текстовые колонки часто оборачивают в функцию `COALESCE`.

    **💡 Живой пример в коде:**
    Склеиваем имя, фамилию и системный код тарифа, чтобы получить красивую строку для отчета:
    
    ```sql
    SELECT 
        PE.VORNAME || ' ' || PE.NAME AS full_name,
        'Тарифный план: ' || GK.GP_VARIANTE AS tariff_info
    FROM S080TGL.GRUNDKONTO GK
    JOIN S080TGL.T_PERSON PE ON PE.PERS_NR = GK.PERS_NR;
    ```

=== "COALESCE — Замена NULL"
    <font size="5">**Обработка пустоты (NULL) в PostgreSQL**</font>

    Возвращает первое значение из списка, которое **не является** `NULL`. Используется для подстраховки, чтобы заменять пустые ячейки в отчете на нормальные дефолтные значения (например, на `0` или текст `Нет данных`).
    
    **Примеры базового синтаксиса:**
    
    * `COALESCE(s.cnt, 0)` — если в подсчетах пусто (`NULL`), нарисует красивый `0`.
    * `COALESCE(name, 'Без имени')` — если у клиента не заполнено имя, выведет текст-заглушку.

    **💡 Живой пример в коде:**
    Выводим список всех продуктов из справочника. Если по какому-то продукту за неделю не было открыто ни одного счета, `LEFT JOIN` вернет пустоту (`NULL`), а мы ловко превратим её в ноль:
    
    ```sql
    SELECT 
        p.GP_VARIANTE,
        COALESCE(COUNT(gk.EROEFFNUNGSDATUM), 0) AS total_opened
    FROM S080TGL.PRODUCT_TABLE p
    LEFT JOIN S080TGL.GRUNDKONTO gk ON gk.GP_VARIANTE = p.GP_VARIANTE
    GROUP BY p.GP_VARIANTE;
    ```

=== "FILTER — Выборочный подсчет"
    <font size="5">**Фильтрация внутри агрегатных функций в PostgreSQL**</font>
    
    Позволяет применить условие `WHERE` конкретно к одному счетчику, не отсекая при этом строки из всего остального запроса. Мега-удобно, когда в одной таблице нужно сделать несколько разных счетчиков (например, общий итог и итог только за неделю).
    
    **Примеры базового синтаксиса:**
    
    * `COUNT(*) FILTER (WHERE статус = 'активен')` — посчитает только активные строки.
    * `SUM(сумма) FILTER (WHERE дата >= '2026-01-01')` — сложит суммы только за 2026 год.

    **💡 Живой пример в коде:**
    Считаем общее количество открытых счетов по каждому тарифу, но в соседней колонке ловко выводим только те счета, которые были открыты за последние 7 дней:
    
    ```sql
    SELECT 
        GK.GP_VARIANTE,
        -- Счетчик 1: Считает абсолютно все записи в группе
        COUNT(GK.EROEFFNUNGSDATUM) AS total_cnt,
        
        -- Счетчик 2: Сито, считает только записи за последние 7 дней
        COUNT(GK.EROEFFNUNGSDATUM) FILTER (
            WHERE GK.EROEFFNUNGSDATUM >= CURRENT_DATE - INTERVAL '7 days'
        ) AS last_week_cnt
    FROM S080TGL.GRUNDKONTO GK
    GROUP BY GK.GP_VARIANTE;
    ```
